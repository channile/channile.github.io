26.删除有序数组中的重复项

分析

题目要求：
我们 原地 删除重复出现的元素，使每个元素 出现一次， 并且返回删除后数组的新长度。
不要使用额外的数组空间，你必须在原地修改输入数组并在使用 O(1) 额外空间的条件下完成。

O(1):当一个算法的空间复杂度为一个常量，即不随被处理数据量n的大小而改变时，可表示为O(1)。

想法

既然要求使用O(1)额外空间，那就不能再创建新数组，所以我们只能使用双指针。

我一开始是想正判断,但是发现这样做只能满足小部分条件，贴一下我的最初代码

但是这样只能满足我假想出来的数组[1,1,1,2],除此之外对其他数组并不成立。

于是换个方向，判断nums[j] != nums[i]，答案出来了。

class Solution {
public int removeElement(int[] nums, int val) {
int i = 0;
for (int j = 0; j < nums.length; j++) {
if (nums[j] != val) {

            //这里是和上一题不一样的地方
            nums[i] = nums[j];
            i++;
        }
    }
    return i;
}

}

```
就这样吧，但是要注意和上一题解法不一样的是i++和nums[i] = nums[j]要交换一下位置，是为了让nums[i]先得到nums[j]的值，再将i加1

总结

我们从这类问题中，要学会从反方向思考问题。
在这种题目中正解比较难解，所以我们可以从反思维入手来思考思考问题。

当 nums[j] == nums[i] 时，i=0且j++，那么当出现第一次 nums[j] != nums[i] 时，i=1 并且把此时 nums[j] 的值赋值给 nums[i]即nums[1]
当循环结束时，由于数组是从0开始，所以 i 需要+1才能返回数组的正确长度

同理，当 nums[j] == val 也是同样的解法。
